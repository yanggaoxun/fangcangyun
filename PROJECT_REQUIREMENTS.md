# 菌菇类方舱自动化后台管理系统需求文档

## 项目概述
基于 Laravel 12 + Filament 4.0 框架开发的菌菇类方舱自动化后台管理系统，用于管理和监控蘑菇 cultivation chambers（ cultivation 方舱）的自动化生产过程。

## 核心功能模块

### 1. 基地管理模块
**功能描述**：管理不同地理位置的基地信息

**已实现功能**：
- 基地信息管理（增删改查）
- 基地基础信息：编号、名称、位置、负责人、联系电话
- 基地状态管理：正常运营、暂停使用、维护中
- 基地与菌种库存关联：每个基地管理自己的菌种库存

**数据库表结构**：
- `bases` 表：存储基地信息
- `base_strain_stocks` 表：存储各基地的菌种库存
- 字段：code（基地编号）、name（基地名称）、location（位置）、manager（负责人）、phone（联系电话）、description（描述）、status（状态）

### 2. 方舱管理模块
**功能描述**：管理 cultivation 方舱的基础信息和环境参数

**已实现功能**：
- 方舱基础信息管理：编号、名称、位置、容量、状态
- 方舱与基地关联：每个方舱属于一个基地
- 方舱状态：空闲、种植中、维护中
- 方舱与环境数据关联：接收方舱上传的环境数据

**数据库表结构**：
- `chambers` 表：存储方舱信息
- 字段：code（方舱编号）、name（名称）、base_id（所属基地）、capacity（容量）、status（状态）、description（描述）

### 3. 菌种菌包管理模块
**功能描述**：管理菌种信息和菌包批次

**已实现功能**：
- 菌种信息管理：以菌种类别作为名称（平菇、香菇、金针菇、其他）
- 菌种环境参数：温度、湿度、CO2、pH范围设置
- 基地菌种库存：每个基地独立管理菌种库存
- 菌包批次管理：批次编号、接种日期、预计收获日期、数量统计
- 库存扣减逻辑：创建批次时自动扣减对应基地的菌种库存

**数据库表结构**：
- `mushroom_strains` 表：存储菌种信息
- `base_strain_stocks` 表：存储基地菌种库存（base_id + strain_id唯一）
- `batches` 表：存储菌包批次信息
- 关联关系：批次通过chamber关联基地，通过strain_id关联菌种

**关键业务逻辑**：
- 新建菌种时可配置多个基地的初始库存
- 创建批次时选择基地，只显示该基地有库存的菌种
- 批次创建成功后自动扣减对应基地的菌种库存
- 库存不足时禁止创建批次并提示错误

### 4. 环境监控模块
**功能描述**：实时监控方舱内环境参数

**已实现功能**：
- API数据接收：方舱通过API上传环境数据（温度、湿度、CO2、pH、光照、土壤湿度）
- 只读数据展示：Filament后台只读显示环境数据，不允许手动修改
- 数据筛选功能：按基地、方舱、时间范围、异常状态筛选
- 异常标记：支持标记异常数据
- 历史数据记录：支持按时间范围查询

**数据库表结构**：
- `environment_data` 表：存储环境数据
- 字段：chamber_id（方舱ID）、temperature（温度）、humidity（湿度）、co2_level（CO2浓度）、ph_level（pH值）、light_intensity（光照强度）、soil_moisture（土壤湿度）、recorded_at（记录时间）、is_anomaly（是否异常）

**API接口**：
- `POST /api/v1/environment-data` - 单条数据上传
- `POST /api/v1/environment-data/batch` - 批量数据上传
- 参数：chamber_code, temperature, humidity, co2_level, ph_level, light_intensity, soil_moisture, recorded_at, is_anomaly, notes

### 5. 设备控制模块
**功能描述**：管理方舱内的自动化设备

**已实现功能**：
- 设备信息管理：传感器、控制器、执行器等设备档案
- 设备状态监控：在线状态、运行参数
- 远程控制功能：支持远程调节环境控制设备
- 设备故障报警：设备异常时及时通知

**数据库表结构**：
- `devices` 表：存储设备信息
- `device_controls` 表：存储设备控制记录
- 字段：设备类型、状态、参数设置等

### 6. 告警管理模块
**功能描述**：系统告警和异常处理

**已实现功能**：
- 多级告警机制：紧急、重要、一般三级告警
- 告警类型：环境异常、设备故障、系统异常
- 告警通知：系统内通知
- 告警处理流程：记录处理过程和结果

**数据库表结构**：
- `alerts` 表：存储告警信息
- 字段：chamber_id（方舱ID）、type（告警类型）、level（告警级别）、message（告警消息）、status（状态）

## 数据流转流程

### 环境数据流
1. 方舱设备通过API定期上传环境数据
2. 系统接收并存储到environment_data表
3. 后台管理界面只读显示环境数据
4. 支持按基地、方舱、时间筛选查看

### 菌种库存流
1. 管理员在菌种管理中添加菌种，配置各基地初始库存
2. 创建种植批次时选择基地和菌种
3. 系统自动扣减对应基地的菌种库存
4. 库存不足时阻止批次创建

### 批次管理流
1. 选择基地 → 系统自动筛选该基地可用的菌种
2. 选择菌种 → 显示该基地库存提示
3. 输入数量 → 系统验证库存是否充足
4. 提交创建 → 扣减库存、更新方舱状态为"种植中"
